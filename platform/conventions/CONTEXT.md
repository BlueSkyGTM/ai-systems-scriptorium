# CONTEXT — platform/conventions/

The writing law for every book.

## What's here

- `STYLE.md` — the **Style Contract** (Zinsser, treated as an engineering spec). This is **Tier-1
  policy** per the precedence rule in `CLAUDE.md`/`AGENTS.md`: a local `CONTEXT.md` cannot override
  it. If a draft and the contract disagree, the contract wins. Run it on every lesson before ship.
- `AUTHORING.md` — the curriculum the authoring agent follows: the three-source rule (ingredient +
  Microsoft Learn + seam framing), the input→output map, what to load.

## Load / don't-load

- **Load:** both files, before authoring or verifying a single lesson — `AUTHORING.md` first, then
  `STYLE.md`.
- **Do NOT load:** book internals or the vault here; this is the law, not the material.

## Handoff

Back to `platform/pipeline/CONTEXT.md` (AUTHOR/VERIFY apply this law). No gates own this folder.
