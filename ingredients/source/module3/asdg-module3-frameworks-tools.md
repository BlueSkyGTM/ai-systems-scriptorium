### 01 · LangChain Deep Dive  [seam-relevant]
LangChain is a modular ecosystem for building production-grade LLM applications, organized into three tiers: `langchain-core` (minimal abstractions), partner packages (provider integrations), and LangGraph (stateful orchestration). LCEL (LangChain Expression Language) uses the pipe (`|`) operator to build composable DAGs of execution with automatic async, parallelism, and LangSmith observability. The v1.0 release standardized JSON Schema compatibility across validation libraries (Pydantic v2, Zod 4, Valibot, ArkType, TypedDict) for `with_structured_output()`, `bind_tools()`, and `@tool`, decoupling framework choice from validator choice. Production systems should depend on `langchain-core` plus selected partner packages and LangGraph, avoiding `langchain-community` and legacy classes (`LLMChain`, `RetrievalQA`, `AgentExecutor`) now frozen in `langchain-classic`.
**Key concepts:** LCEL, Runnables, DAG execution, `langchain-core`, Partner Packages, `langchain-community`, `langchain-classic`, LangGraph, `with_structured_output()`, `bind_tools()`, `@tool`, JSON Schema, Pydantic v2, Zod 4, Valibot, ArkType, TypedDict, MCP (Model Context Protocol), async-by-default, parallelism, LangSmith observability, package tiering.

### 02 · LangGraph Orchestration  [seam-relevant]
LangGraph is a graph-based runtime and the de facto standard for building stateful, multi-agent systems, reaching v1.0 in late 2025. Unlike standard acyclic chains, it natively supports cycles, state persistence, and human-in-the-loop interventions, enabling patterns like ReAct where agents can loop back after tool errors. The architecture maps agents as graphs composed of nodes (Python functions), edges (paths), and conditional edges (logic gates driven by a centralized state schema). It provides thread-based persistence with checkpointing for time-travel debugging and supports multi-agent topologies including supervisor, peer-to-peer, and hierarchical patterns.
**Key concepts:** Nodes, Edges, Conditional Edges, State Schema, Cyclic vs. Acyclic Workflows, Annotated Reducers (e.g. `add_messages`), Thread-based Persistence, Checkpointing, Time-Travel, Supervisor Pattern, Peer-to-Peer Pattern, Hierarchical Pattern (Nested Graphs), State Narrowing, Trim Runnables, White Box Framework, Open Source / On-Prem Portability.

### 03 · LangSmith Observability  [seam-relevant]
LangSmith is the LangChain-native observability and evaluation platform that provides full trajectory debugging, automated evaluation pipelines, and A/B testing for LLM applications. It automatically captures every node in a LangGraph or Chain execution, enabling trace-level inspection with metadata tagging (e.g., `user_id`, `model_tier`, `is_canary`) and interactive prompt replay for debugging without re-running the entire application. The platform supports gold standard datasets, LLM-as-judge automated evaluators (scoring on categories like Tone, Accuracy, and Safe Action execution), and experiment branching for controlled deployment of system prompt changes with automatic rollback on threshold breaches. It sits in a competitive LLMOps layer alongside Langfuse, LangWatch, Braintrust, and Arize Phoenix.

**Key concepts:** Observability Pyramid (Success Rate → Node Latency/Cost → Raw Traces), Tracing and Trajectories, Metadata Tagging, Gold Standard Datasets, Correction Datasets, LLM-as-Judge, Custom Evaluators (regex, JSON schema, Toxicity), A/B Testing, Experiment Branching, Trace Attribution, Line of Reasoning, Targeted Optimization, Meandering agents.

### 04 · LlamaIndex  [seam-relevant]
LlamaIndex is a data-centric AI framework that has evolved from a RAG library into a comprehensive system for agentic data manipulation and event-driven workflows. Built on the philosophy that data is more important than the model, it structures information into rich metadata "Nodes" and leverages advanced retrievers (Summary, Knowledge Graph, Tree, Keyword) and indexing techniques like Property Graphs and context-aware splitters. Its modern architecture revolves around "Workflows," an event-driven application framework where computational steps communicate by emitting and consuming typed Events, enabling native asynchronous concurrency, stateful Context management, and large-scale parallel document processing. For enterprise deployments, it offers managed ingestion via LlamaCloud and LlamaParse, which utilize Vision-LLMs to understand document layouts, while its query engines can be wrapped as tools for external agents (such as LangGraph).
**Key concepts:** Data-Centric AI, Nodes, Retrievers, Event-Driven Workflows, Typed Events, Context, Property Graphs, Context-Aware Splitters, Dynamic Pathing, LlamaCloud, Managed Ingestion, Parsing as a Model, Agents as Tools, Concurrency, Human-in-the-loop.

### 05 · DSPy: Programming Language Models  [optimize-territory]
DSPy is a framework that shifts LLM application development from manual "Prompt Engineering" to automated "Prompt Compilation," consistently delivering a 10-40% quality lift over hand-tuned prompts. It treats LLM applications like neural networks through three core abstractions: **Modules** (reusable logic blocks like `ChainOfThought`), **Signatures** (declarative, model-agnostic input-output specifications), and **Optimizers** (processes that find the best prompt weights based on a metric). The flagship optimizer, **MIPROv2**, leverages an assistant model for instruction proposal, followed by Bayesian optimization against a training set to select the highest-scoring prompt. DSPy also provides built-in **Assertions** for automated constraint enforcement and allows systems to adapt to underlying model weight updates by simply recompiling, though it introduces compilation latency and upfront costs.
**Key concepts:** Prompt Compilation, Module, Signature, Model-Agnostic, Optimizer, MIPROv2, Instruction Proposal, Bayesian Optimization, Hard and Soft Assertions (`dspy.Suggest`, `dspy.Assert`), Model Drift, Re-compilation, Compilation Latency.

### 06 · Semantic Kernel  [seam-relevant]
Semantic Kernel (SK) is Microsoft's enterprise-grade AI orchestration engine, serving as the primary bridge for organizations committed to the Azure/Microsoft ecosystem and C#/.NET architectures. The framework relies on a Plugin-based model where individual Kernel Functions (either native code or LLM prompts) are registered with the Kernel, allowing the LLM to securely trigger tools via standard function calling. SK leverages enterprise design patterns like dependency injection, strong typing, and deep Azure Active Directory (Microsoft Entra ID) integration to prevent the "spaghetti code" often associated with scaling large AI projects. Much of its forward momentum now ships inside the consolidated Microsoft Agent Framework (successor to AutoGen + SK, GA Q2 2026), though SK remains the foundational orchestration layer for banks and Fortune 500s.
**Key concepts:** Enterprise AI orchestration, C#/.NET architecture, Microsoft Agent Framework, Dependency Injection, Strong Typing, Azure Active Directory integration, Plugins, Kernel Functions, Function Calling, Automatic Intent Detection, Hierarchical Planners, Memory and Connectors, Vector Store Abstraction, Multi-Language Support, Shared prompt templates (.yaml).

### 08 · Framework Selection Guide  [seam-relevant]

This guide provides a decision matrix for selecting AI agent frameworks and SDKs based on production requirements, team expertise, and system scale. The framework landscape is categorized into orchestration frameworks (LangGraph, DSPy, LlamaIndex, CrewAI, MS Agent Framework), lab-specific Agent SDKs (Claude, OpenAI, Google ADK), and coding agents (Claude Code, Cursor, OpenHands). Key architectural principles include resisting framework bloat, using interoperability protocols (MCP for agent-to-tool, A2A for agent-to-agent) to mitigate vendor lock-in, and separating the orchestration layer from the model layer.

**Key concepts:** Framework tiers (L1 Core, L2 Data/IDE, L3 App), Decision Matrix, Build vs. Buy vs. Framework, Anti-patterns (Framework Tunnelling, Golden Hammer, Ignoring Observability), Agent SDKs, Coding Agents, Interoperability Protocols (MCP, A2A), Vendor lock-in mitigation, DSPy Signatures, Universal Model Gateway.

### 09 · Claude Code: The Autonomous Coding Agent  [seam-relevant]
Claude Code is Anthropic's terminal-native autonomous coding agent that operates as a full-stack software engineer, capable of reading codebases, editing files, running commands, and verifying tests until a task is complete. Unlike IDE completion plugins (e.g., Copilot), it autonomously plans and implements entire workflows using its native tools: `bash` for persistent shell execution, `text_editor` for surgical file edits, and `computer` for optional GUI automation. To align the agent with project standards, it relies heavily on the `CLAUDE.md` manifest pattern to inject persistent context, and supports advanced scaling through sub-agent parallelism and Model Context Protocol (MCP) integrations. Safety and production-readiness are enforced via a layered permission model and headless execution, enabling automated, sandboxed operations in CI/CD pipelines.
**Key concepts:** CLI autonomous agent, Extended Thinking, native tools (bash, text_editor, computer), CLAUDE.md manifest, headless mode, Python SDK, sub-agent dispatch, MCP integration, layered permission model, CI/CD pipeline integration.

### 11 · Pydantic AI and Mastra: Typed Agent Frameworks (2026)  [seam-relevant]
Pydantic AI and Mastra are typed agent frameworks that reject untyped prompt-stitching by defining agent loops and tool calls directly in code with strict type enforcement. Pydantic AI targets Python services (especially FastAPI), enforcing Pydantic models end-to-end for HTTP boundaries, tool execution, and LLM outputs, while offering provider portability and native OpenTelemetry tracing. Mastra targets TypeScript-first environments (Next.js/Node), offering full type inference via Zod schemas, built-in local developer playgrounds (`mastra dev`), workflow engines with suspend/resume capabilities, and one-command deployment to popular cloud platforms. Both frameworks trade the large integration ecosystems of older tools like LangGraph for tighter type guarantees, easier unit testing, and structured evaluation capabilities.
**Key concepts:** Typed agent loops, Pydantic AI, Mastra, Python type safety, TypeScript-first agents, Zod schema inference, provider portability, dependency injection, local dev server (`mastra dev`), declarative eval suites, suspend/resume workflows, OpenTelemetry tracing, structured outputs.
