### 01 · Agent Fundamentals [GAP 2: complexity ladder]
Agents are LLM-powered closed-loop reasoning systems defined by the formula `Agent = Reasoning Model + Tool Use + Persistent Memory + Environment Feedback`, moving beyond simple chat wrappers toward integrated Systems with baked-in "System 2" thinking. System design requires choosing between fast System 1 models for routing and UI, versus slow System 2 models for decision gates and complex planning, with agency classified across five levels from L0 scripted chains to L4 ambient agents. Core components include the reasoning model (executive), tools/limbs (standardized via MCP), and memory (short-term context and long-term vector storage), operating through a lifecycle of intake, decomposition, execution, reflection, and completion. Key design challenges include path reliability (preventing infinite loops via chain-of-thought) and agentic drift (mitigated through goal anchoring and secondary observer models).

**Key concepts:** Closed-Loop Reasoning Systems, Agent Formula, System 1 vs. System 2 Thinking, Agency Levels (L0–L4), Reasoning Model, Tool Use, Model Context Protocol (MCP), Short-term/Long-term Memory, Agent Lifecycle (Intake, Decomposition, Execution, Reflection, Completion), Chain-of-Thought (CoT), Path Reliability, Agentic Drift, Goal Anchoring, Secondary Observer Model.

### 02 · Reasoning Loops: ReAct and Beyond [GAP 2: complexity ladder]

Reasoning loops define the control flow of an agent, evolving from the classic ReAct pattern (interleaving thought and action) to more sophisticated approaches like Self-Reflexion (adding a critic/evaluator step with short-term reflection memory), Plan-and-Solve (static planning followed by execution with full re-planning on failure), and Flow Engineering (cyclic state-machine graphs where each node is a specialized prompt or tool). ReAct is identified as fragile—prone to repeating failed actions—leading modern loops to inject negative constraints. Inference-Time Scaling uses internal search trees (e.g., Monte Carlo Tree Search) to simulate action paths before committing, reducing real-world tool calls. The agent itself is redefined as the graph execution engine rather than just the LLM. When choosing architecture, ReAct suits exploratory tasks in unpredictable environments while Plan-and-Solve is preferred for predictable but complex workflows where planning prevents meandering and enables parallelization.

**Key concepts:** ReAct (Thought-Action-Observation loop), Self-Reflexion (Critic step + reflection memory), Plan-and-Solve (Planner/Executor/Re-planner), Flow Engineering (cyclic state-machine graphs, micro-agents), Inference-Time Scaling (internal search trees), negative constraints, stochastic errors, exploratory vs. predictable task routing.

### 03 · Tool Use and MCP [THREAD: safety]
Tools function as the "hands" of an AI agent, with the industry standardizing on the Model Context Protocol (MCP) to provide a unified, local-first communication layer replacing fragmented custom definitions. MCP operates on a client-server architecture where standalone processes expose tools, resources, and prompts via JSON-RPC, enabling strict type validation, dynamic discovery, and security sandboxes. For cross-vendor coordination, the Agent-to-Agent (A2A) protocol complements MCP by enabling orchestrator agents to delegate tasks to specialist agents regardless of their underlying framework. Production environments enforce rigorous defense-in-depth pipelines for these protocols—encompassing OAuth 2.1 authentication, containerized STDIO servers, output validation, and human-in-the-loop (HITL) requirements for dangerous tools—to mitigate vulnerabilities like prompt injection and unauthorized host access.
**Key concepts:** Model Context Protocol (MCP), Agent-to-Agent Protocol (A2A), Agent Communication Protocol (ACP), tool-use mechanism, dynamic manifests, MCP Client/Server architecture, JSON-RPC, Streamable HTTP transport, OAuth 2.1 authorization, high-precision tools, partial tool speculation, MCP Server Cards, MCP Apps, A2A Task Lifecycle, Agent Cards, defense-in-depth checklist, security sandboxing, prompt injection detection, computer-use tools, Context7

### 04 · Multi-Agent Orchestration
Complex enterprise systems are transitioning from single agents to coordinated teams of specialized agents managed via orchestration patterns like hierarchical supervisors, swarms, and graph-based workflows. Graph-based orchestration has emerged as the dominant 2026 paradigm due to its explicit control flow, visualizability, and state-awareness, while the Agent-to-Agent (A2A) protocol now enables cross-vendor networks via Agent Cards and asynchronous task delegation. Effective multi-agent deployment requires managing state across local and global contexts (addressing write conflicts via transactional handoffs) and selecting the appropriate architectural topology—such as peer-to-peer debate for high-accuracy tasks or pipelines for linear workflows.
**Key concepts:** Hierarchical Supervisor Pattern, Swarms/Handoffs, Graph-Based Orchestration, Cross-Vendor Agent Networks, Agent-to-Agent (A2A) Protocol, Agent Cards, Shared Blackboard, Transactional Handoffs, Agentic Debate, Cognitive Load, Paperclip Pattern.

### 05 · Agent Memory and State

Agent memory has evolved into a multi-tiered cognitive architecture consisting of four layers—Working (L1), Episodic (L2), Semantic (L3), and Procedural (L4)—each defined by distinct read/write patterns, latency budgets, and storage technologies. Determining the correct tier for a given piece of information is a critical architecture decision, as misplaced facts lead to predictable scaling failures, elevated latency costs, and compounding storage bloat over time. Production systems must also navigate severe failure modes such as memory poisoning, cross-tenant leakage, and hallucinated writes, requiring strict isolation boundaries, provenance tracking, and threshold-based consolidation policies. As agents evolve, frameworks like Mem0, Zep/Graphiti, and Anthropic's filesystem-mounted solutions implement these governance postures, handling everything from bitemporal fact tracking to scheduled reflection and skill updates.

**Key concepts:** Multi-tiered cognitive architecture (Working, Episodic, Semantic, Procedural), KV-cache tiling, self-search, bitemporal knowledge graphs, graduation rule (consolidation thresholds), memory poisoning (MINJA/MemoryGraft), Day-30 drift, HaluMem (hallucinated memory writes), cross-tenant isolation, provenance tags, TTT-E2E (test-time-training)

### 06 · Planning and Decomposition [GAP 2: complexity ladder]

Planning serves as the "System 2" component that prevents agents from wandering during multi-stage problem-solving, spanning a complexity spectrum from simple linear steps through branching and hierarchical decomposition to search-based methods. Production systems increasingly rely on reasoning-native models to perform planning internally via inference-scaling, generating a mental draft that is translated into a Task DAG for execution. Dynamic (adaptive) planning with checkpointed state persistence is preferred over static plan-and-solve for robustness, while recursive task decomposition uses sub-agent spawning with minimal context to prevent token bloat, and Monte Carlo Tree Search (MCTS) enables high-stakes decisions through simulated tool-call paths scored by a reward model.

**Key concepts:** Planning Spectrum (Linear, Branching, Hierarchical, Search-Based), Static Plan-and-Solve, Dynamic Adaptive Planning, Checkpointed Planning, Chain-of-Thought reasoning, Hidden Planner, Inference Scaling, Task DAG, Recursive Task Decomposition, Sub-Agent Spawning, Minimal Context principle, Monte Carlo Tree Search (MCTS), Reward Model, Decomposition Depth Limits, Granularity Checks, Global Controller, Context Re-evaluation, Reasoning Density.

### 07 · Error Handling and Recovery [THREAD: safety]

Agent failures are non-deterministic and span four categories: hallucinated tools, schema violations, environment errors, and logical stalls (the ReAct Loop of Death). Modern error handling replaces traditional try/catch blocks with **agentic self-correction**, where error messages are fed back to the model as tokens of information so it can reason around failures and fix the plan rather than merely fixing code. Stateful rollbacks (checkpointing) save state snapshots after every successful tool call, enabling a supervisor agent to reset to the last known safe state and force an alternative path. Additional safeguards include counter-based intervention that interrupts after repeated identical tool calls and injects a mandatory pivot instruction, graceful degradation to simpler agents or RAG-only mode, and output validation agents that detect silent failures where tools return 200 OK with incorrect data.

**Key concepts:** Agentic Self-Correction, Error Injection, Synthesized Observation, Stateful Rollbacks, Checkpointing, Counter-Based Intervention, Pivot Instruction, Graceful Degradation, Output Validation Agents, Verifier Agents, Taxonomy of Agent Failures, Hallucinated Tools, Schema Violation, Environment Error, Logical Stall, ReAct Loop of Death.

### 08 · Human-in-the-Loop Patterns [THREAD: safety]
Human-in-the-Loop (HITL) mechanisms bridge the reliability gap in agentic systems by ensuring human oversight in high-stakes environments, spanning a spectrum from human-in-command (low agent autonomy) to human-on-the-cycle (post-completion audit only). Modern frameworks such as LangGraph and Microsoft Agent Framework expose deterministic interrupts and breakpoints that freeze agent state prior to sensitive tool calls, awaiting explicit approve or reject signals. Advanced patterns include time-travel debugging via state injection, co-reasoning through shared scratchpads where humans act as partners rather than judges, and confidence-based escalation that auto-pauses on high uncertainty scores derived from model logprobs. To prevent alert fatigue and over-reliance, production systems employ threshold tuning, contextual summaries, forced review checkpoints, and synthetic error injections.
**Key concepts:** HITL Spectrum, Deterministic Breakpoints, Interrupts, State Preservation, Time-Travel Debugging, State Injection, Co-Reasoning, Shared Scratchpads, Confidence-Based Escalation, Uncertainty Score, Threshold Tuning, Contextual Summary, Forced Review Checkpoints, Synthetic Error Injections, Over-Reliance risk.

### 09 · Agentic Security and Sandboxing [THREAD: safety]

Agents expand the security attack surface from information leakage to direct action execution, creating risks such as data exfiltration, financial loss, and infrastructure damage. Core defenses include **Action Sandboxing** via ephemeral micro-VMs (e.g., E2B, Docker) that are destroyed after each code execution, **Permission Scoping** enforcing least-privilege principles through read-only defaults and token-scoped database roles, and the **Model-in-the-Middle (Proxy) Pattern** where a hardened firewall LLM or policy engine inspects and filters tool calls before execution. **Audit Logging** captures the full deterministic trace (Input → Thought → Call → Result → Interpretation) for SOC2/HIPAA compliance, while parameterized tools and instruction hierarchy (system instructions overriding user instructions) treat agents as untrusted users against protected infrastructure.

**Key concepts:** Action Isolation, The Proxy Pattern, OWASP LLM Top 10 v2.0 agentic risks, excessive agency, data exfiltration, Action Sandboxing, micro-VMs, E2B, Docker-Local, transient environments, Minimum Agency, Least Privilege, Read-Only by Default, Token Scoping, Rate-Limiting Actions, Model-in-the-Middle, Firewall Model, Proxy Agent, Audit Logging, Deterministic Traceability, SOC2/HIPAA compliance, Parameterized Tools, Limited-Scope Role, Row Level Security (RLS), Instruction Hierarchy, System-Priority training, Untrusted User model.

### 10 · Evaluating Agentic Systems [THREAD: safety]
Evaluating agents differs fundamentally from RAG evaluation by focusing on reliability, efficiency, and safety across an entire execution trajectory rather than single-response accuracy. Trajectory benchmarks compare an agent's actual path against an optimal path, with efficiency scored as optimal steps divided by agent steps. Key metrics include Task Success Rate (requiring correct final state via valid path), Action Success Rate, and Unit Cost per Task, while production teams employ shadow execution and LLM-as-judge techniques to assess step quality and guide behavior alignment.
**Key concepts:** Trajectory evaluation, Task Success Rate (TSR), Action Success Rate (ASR), Unit Cost per Task, LLM-as-Judge, Shadow Execution, Step Budgets, Mock Environments, State-Based Verification, SWE-bench, WebArena, GAIA, Direct Preference Optimization (DPO).
