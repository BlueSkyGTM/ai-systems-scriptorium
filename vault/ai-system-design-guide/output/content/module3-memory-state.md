### 01 · Memory Architectures

LLM memory has evolved from simple history buffers into a three-tiered cognitive architecture (L1-L3) that mirrors human cognitive systems to balance speed, cost, and recall capacity. L1 (Working Memory) utilizes the active context window and KV cache with strategies like sliding windows and prefix caching for sub-50ms latency. L2 (Episodic Memory) stores session histories and past experiences in vector databases for semantic search retrieval at 100-300ms latency, while L3 (Semantic Memory) holds immutable facts and learned rules in knowledge graphs or managed services like Mem0 at higher latency. Production stacks increasingly rely on dedicated memory layers (Mem0, Zep, Letta, Cognee) over vector stores, where memories move between tiers via consolidation patterns involving extraction, indexing, and decay, with strict namespace sharding and RLS to prevent privacy leakage.

**Key concepts:** Three-Tiered Cognitive Architecture (L1-L3), Working Memory (Context Window, KV Cache, Sliding Windows, Prefix Caching), Episodic Memory (Vector DBs, Experience Replay), Semantic Memory (Knowledge Graphs, Mem0, Truth Anchoring), Memory Consolidation (Extraction, Indexing, Decay), Namespace Sharding, Row Level Security (RLS), PII-Scrubbing, Lost in the Middle, Strategic Retrieval.

### 02 · Short-Term Context Management

Short-term context (L1 Memory) is the high-speed interface where reasoning happens, managed through KV Cache Optimization and Dynamic Context Allocation rather than simple message lists. The context lifecycle proceeds through intake, GPU-based processing of KV caches, and eviction of old tokens. Modern inference engines like vLLM and TensorRT-LLM use PagedAttention to break memory into blocks, reducing fragmentation by 60-80% and enabling larger batch sizes. Persistent prefix caching preserves the KV cache for static prompt portions (system prompts, tool schemas) so compute is only spent on new, dynamic content. Context management strategies include sliding windows, summarization, hybrid approaches, and contextual compression techniques such as selective dropping of irrelevant thought blocks and token pruning via smaller models.

**Key concepts:** L1 Memory, KV Cache Optimization, Dynamic Context Allocation, Context Lifecycle (Intake, Processing, Eviction), PagedAttention, KV Cache Tiling, memory fragmentation, Persistent Prefix Caching, Sliding Window, Summarization, Hybrid context strategy, Contextual Compression, Prompt Hardening, Selective Dropping, Token Pruning, Model Context Window vs. Application Context Window, Buffer Zone, static-first prompt design.

### 03 · Long-Term Memory

Long-term memory (L2 & L3) provides persistence across sessions by combining Vector, Graph, and Relational data into Multi-Representation Stores. Dedicated memory services such as Zep, Mem0, Letta, and Cognee wrap these stores with built-in conversation summarization, entity extraction, and temporal awareness. It is bifurcated into episodic memory, which stores interaction trajectories (summaries plus embeddings and raw logs), and semantic memory, which stores discovered facts as entity-relationship triplets within knowledge graphs (Neo4j, AWS Neptune). Staff-level engineers implement Hybrid Vector-Graph Storage (GraphRAG-style memory), where vector search finds related nodes and graph traversal finds structurally connected nodes. Memory is managed through temporal decay (scoring relevance by recency and access frequency), consolidation (merging related interactions into single high-quality summary nodes), and explicit forgetting for GDPR compliance, while `user_id` serves as a hard partition key in vector DB metadata to prevent cross-session leakage.

**Key concepts:** Long-Term Memory (L2 & L3), Multi-Representation Stores, Episodic Memory, Trajectories, Cold Storage, Semantic Memory, Fact Extraction Agent, Knowledge Graphs, Hybrid Vector-Graph Storage (GraphRAG-style Memory), Temporal Decay, Consolidation, Explicit Forgetting, Cross-Session Leakage, Hard Partition Keys, Catastrophic Forgetting (Index Overload), Quality-Weighted Retrieval.

### 04 · Agentic Memory with Mem0

Mem0 (alongside peers Zep, Letta, and Cognee) represents the shift from passive logs to Active Memory, automatically digesting conversations to create a persistent, evolving user profile that enhances personalization across interactions. The system operates through a Digest Loop spanning memory tiers: the agent observes conversations in L1, a background Memory Agent extracts memorable facts, compares them against existing records in L3, and merges or updates records with new timestamps upon conflicts. Mem0 supports recursive, self-updating memories via Periodic Reflection—running daily jobs to review active Goal Nodes and generate Proactive Reminders—and integrates with state-machine architectures like LangGraph as an External State Provider that injects user preferences into the global reasoning state. At enterprise scale, it manages consistency across platforms (Web, Mobile, Slack) and reduces friction by avoiding repetitive qualifying questions. Key mechanisms include deduplication, entity linking, cross-session synchronization, temporal weighting, thresholded relevance scoring, negative retrieval, and automatic memory pruning of low-value entries.

**Key concepts:** Active Memory, Mem0, Zep, Letta, Cognee, Digest Loop, L1 (conversation observation), L3 (persistent fact storage), Insights vs. raw logs, Self-Updating Memories, Recursive Memory, Periodic Reflection, Goal Nodes, Proactive Reminders, External State Provider, LangGraph integration, Personalization at Scale, Cross-Session Consistency, Deduplication, Entity Linking, Conflicting Identity Resolution, Cross-Session Synchronization, Temporal Weighting, Memory Fatigue, Thresholded Relevance, Relevance Score, Negative Retrieval, Memory Pruning.

### 05 · Semantic Caching

Semantic caching evolves beyond exact string matching by using query embedding vectors and cosine similarity thresholds to identify and reuse completions for "equivalent" queries, reducing costs by 30-70% and cutting latency from seconds to milliseconds. The standard stack relies on RedisVL or GPTCache for low-latency vector search, with dynamic TTLs ensuring popular answers persist while stale data is evicted. A critical risk is **Semantic Drift**—returning incorrect results when thresholds are too loose—which is mitigated using multi-stage validation pipelines including embedding, nearest-neighbor search, threshold checks (e.g., <0.05 distance), and optional LLM verifier models for high-stakes queries. At low volumes, semantic caching can be more expensive than raw LLM calls due to the embedding API tax, making it architecture-appropriate only at high scale; multimodal frontier models extend this caching to image and audio queries via visual similarity and audio fingerprinting.

**Key concepts:** Semantic Matching, Exact Cache vs. Semantic Cache, Query embedding vector, Cosine similarity threshold, Embed, Search, Threshold Check, LLM Verification, Verifier Model, RedisVL, GPTCache, Hybrid Caching, Dynamic TTL, Multimodal Semantic Caching, Visual Similarity, Audio Fingerprinting, Semantic Drift, Multi-Stage Validation, Entity-Match check, Threshold Tightening, Embedding Tax, High Scale cache hit rate.

### 06 · State Management Patterns

State management in AI systems has evolved from simple sessions to **Stateful Agent Graphs**, where the "State" object serves as the single source of truth for an agent session, typically implemented as a strictly typed, append-only structure. Industry has converged on **cyclic graphs** (state machines) composed of nodes (functions that process state) and edges (conditional logic that determines transitions), with frameworks like LangGraph becoming the default control-flow runtime. Production resilience is achieved through **checkpointing**, where every state update persists to a database (Postgres/Redis), enabling crash recovery, asynchronous agent UX, and time-travel capabilities for human-in-the-loop intervention. Advanced patterns include **parallel state** (fork/join via fan-out to sub-agents and fan-in merging), state pruning, and message summarization to prevent state bloat and token-limit exhaustion in long-running sessions.

**Key concepts:** Stateful Agent Graphs, State Object (Single Source of Truth), Strictly Typed state, Append-Only state, Cyclic Graphs, Nodes, Edges, Conditional Logic, Checkpointing, Persistence Layer, Asynchronous Agents, Parallel State, Fork/Join, Fan-out, Fan-in, Time-Travel, State Rewriting, State Pruning, Message Summarization, LangGraph.
