## 1. Problem Statement

Customer support tickets arrive untagged into a shared inbox. Routing them by hand
costs roughly 30 seconds per ticket and misroutes about 1 in 8. The goal of this
artifact is to ship a **tiny, auditable classifier** that:

1. Beats a naive majority-class baseline by a meaningful margin (at least 5pp on both
   exact-match accuracy and macro-F1).
2. Refuses to ship if it can't prove it: the eval script exits non-zero and blocks
   downstream pipelines.
3. Runs end-to-end on a laptop CPU in seconds, so the whole loop
   (train, eval, gate) fits inside a CI step.

The dataset is a synthetic 5-class problem (`billing`, `bug`, `feature`, `praise`,
`security`) drawn from deterministic templates. The point is not the dataset; the
point is the **discipline**: seed-fixed training, a held-out test set, a baseline the
tuned model must clear, and a metrics trail a reviewer can inspect.

## 2. Approach

- **Model:** [TODO: one-line architecture, e.g. "embedding + mean-pool + 2-layer MLP, ~Nk params"]
- **Training:** PyTorch, Adam, fixed epochs, full-batch on the synthetic set.
  `random.seed(42)`, `torch.manual_seed(42)`.
- **Baseline:** majority class, computed at training time and stored in the checkpoint
  as `train_majority_class`. The gate predicts that class for every input. No sklearn.
- **Eval gate:** exact-match accuracy **and** macro-F1. PASS only if the tuned model
  beats the baseline by at least 5pp on **both**. Exit code 0 = PASS, 1 = BLOCK.
- **Tracking:** MLflow offline, `sqlite:///outputs/mlruns.db` (optional; `--no-mlflow` skips it).
