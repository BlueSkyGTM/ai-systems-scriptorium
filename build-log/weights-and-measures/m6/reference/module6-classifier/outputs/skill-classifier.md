## 1. Problem Statement

Customer support tickets arrive untagged into a shared inbox. Routing them by hand
costs ~30 seconds per ticket and misroutes roughly 1 in 8. The goal of this artifact
is to ship a **tiny, auditable classifier** that:

1. Beats a naive majority-class baseline by a meaningful margin (≥ 5pp on both
   exact-match accuracy and macro-F1).
2. Refuses to ship if it can't prove it — the eval script exits non-zero and blocks
   downstream pipelines.
3. Runs end-to-end on a laptop CPU in under a minute, so the whole loop
   (train → eval → gate) fits inside a CI step.

The dataset is a synthetic 5-class problem (`billing`, `bug`, `feature`, `praise`,
`security`) drawn from deterministic templates. The point is not the dataset; the
point is the **discipline**: seed-fixed training, held-out test set, a baseline that
the tuned model must clear, and an MLflow trail that a reviewer can inspect.

## 2. Approach

- **Model:** [TODO: one-line architecture, e.g. "GloVe-avg + 2-layer MLP, ~120k params"]
- **Training:** PyTorch, AdamW, early stopping on validation exact-match.
  `torch.manual_seed(42)`, `random.seed(42)`, deterministic dataloader.
- **Baseline:** `sklearn.dummy.DummyClassifier(strategy="most_frequent")` — fit on
  the same training split, evaluated on the same held-out test set. This is the
  *only* use of sklearn in the artifact.
- **Eval gate:** exact-match accuracy **and** macro-F1. PASS only if tuned model
  beats baseline by ≥ 5pp on **both**. Exit code 0 = PASS, 1 = BLOCK.
- **Tracking:** MLflow offline, `sqlite:///outputs/mlruns.db`. Every run logs params,
  metrics, and the gate decision.
