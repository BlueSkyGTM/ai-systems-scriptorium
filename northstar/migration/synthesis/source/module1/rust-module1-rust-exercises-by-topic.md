# Module 1 · Rust — Exercises by Topic

> Source: `100-exercises-to-learn-rust` (book narrative + exercises), 8 topics / 98 exercises. Rust is taught as the serving layer, CLI tooling, and concurrency/async substrate. `08_futures` (async) maps directly to concurrent agent tasks and async tool calls.

## Topics

## 01 · Intro

This topic introduces the course structure and Rust's foundational syntax. It covers the development workflow (tooling like `rustup`, IDEs, `wr`), then walks through basic Rust syntax: function definitions with `fn`, type annotations, input parameters, return types, implicit returns, and comments. It also establishes that Rust is statically typed, meaning every value's type must be known at compile-time, enabling the compiler to catch errors before runtime.

**Rust concepts:** `fn` keyword, function signatures, input parameters with type annotations, return types, unit type `()`, implicit returns vs `return` keyword, `&'static str` / `&str` string slices, `String` type, `format!` macro, statically typed language, `assert_eq!` macro, single-line comments (`//`).

**Why it matters (AI Systems Engineering):** This foundation is the entry point to building Rust-based serving layers and CLI tooling, where static typing ensures type-safe schemas and compile-time correctness. Understanding function signatures and type annotations is essential for writing safe, performant Rust services that replace Python in production AI systems.

## 02 · Basic Calculator
This topic covers Rust fundamentals by building a calculator, introducing primitive integer types, variable declaration with type inference, arithmetic and comparison operators, control flow with `if`/`else` expressions, and looping constructs (`while`, `for`). It also addresses critical numeric safety concerns such as panics on division by zero, integer overflow/underflow behavior across dev/release profiles, and explicit type casting with the `as` operator.

**Rust concepts:** primitive integer types (`u8`–`u128`, `i8`–`i128`), `bool`, `let` bindings, explicit type annotations, type inference, immutability by default, `mut` keyword, `fn` definitions, arithmetic operators (`+`, `-`, `*`, `/`, `%`), comparison operators (`==`, `!=`, `<`, `>`, `<=`, `>=`), `if`/`else`/`else if` expressions, `while` loops, `for` loops, ranges (`..`, `..=`), panics (`panic!` macro), integer overflow and underflow, `overflow-checks` profile setting, `wrapping_` methods, `saturating_` methods, `as` type casting and truncation.

**Why it matters (AI Systems Engineering):** Understanding Rust's strict type rules and overflow behavior is essential when building CLI tooling and serving layers that process numeric inputs safely and predictably. Mastery of explicit casting and saturating arithmetic prevents silent data corruption in concurrent agent pipelines or batch-processing tasks where numeric precision directly affects model-serving correctness.

## 03 · Ticket V1
This topic introduces Rust's ownership system through modeling a JIRA-like ticket entity, covering structs, methods, encapsulation via modules and visibility, and the borrow checker's rules for references. It then dives into memory layout (stack vs. heap), the representation of pointers and references, and deterministic destruction via `drop` scopes—showing how Rust achieves memory safety without a garbage collector.
**Rust concepts:** structs, fields, instantiation, methods (`self`, static methods), `impl` blocks, modules (`mod`, `use`, `crate`, `super`), visibility (`pub`, `pub(crate)`, `pub(super)`), encapsulation, constructors, accessor and setter methods, ownership, move semantics, borrowing, immutable and mutable references, stack vs. heap allocation, `usize`, pointers, `std::mem::size_of`, scopes, destructors, `std::mem::drop`.
**Why it matters (AI Systems Engineering):** Ownership and borrowing are the backbone of Rust's safe concurrency and async model—understanding how to define type-safe, encapsulated data structures with controlled mutation is essential for building reliable serving layers and concurrent agent task systems without data races or garbage-collection pauses.

## 04 · Traits
This chapter introduces traits as Rust's mechanism for defining interfaces and shared behavior across types. It covers trait definition and implementation, the orphan rule, operator overloading, derive macros, and generic programming via trait bounds. It then explores key standard library traits including `PartialEq`, `Add`, `From`/`Into`, `Deref`, `Sized`, `Clone`, `Copy`, and `Drop`, along with associated types versus generic trait parameters and string slices (`&str`) as fat pointers.

**Rust concepts:** trait definition and implementation, orphan rule, extension traits, operator overloading via `std::ops` and `std::cmp` traits, default/required methods, derive macros, trait bounds (`where` clauses, inline bounds), generics, `&str` string slices, fat pointers, `Deref` and deref coercion, associated types (`type Target`), marker traits (`Sized`, `Copy`), auto traits, `Sized` and `?Sized`, dynamically sized types, supertraits/subtraits, `From`/`Into` and blanket implementations, `Clone`, bitwise copy semantics, `Drop` trait, destructors.

**Why it matters (AI Systems Engineering):** Traits are the backbone of type-safe abstraction in Rust, enabling generic serving-layer components, composable CLI tooling, and type-safe schemas that enforce contracts at compile time. Understanding standard traits like `Clone`, `From`/`Into`, and `Drop` is essential for writing safe, efficient data transformations and resource management in concurrent and async agent task pipelines.

## 05 · Ticket V2

This topic refines Rust domain modelling by replacing loose `String` types with enums, `Option`, and `Result` to encode domain constraints and fallibility directly into the type system. It covers exhaustive pattern matching, idiomatic error handling using the `Error` trait (and the `thiserror` crate), and fallible conversions via `TryFrom`/`TryInto`, culminating in a design philosophy where invalid states are unrepresentable and values are valid by construction. The sections also introduce Rust's package system (libraries, binaries, crates, and dependencies).

**Rust concepts:** enums, variants with data (struct-like and tuple-like), `match`, exhaustiveness, catch-all patterns, `if let`, `let/else`, `Option<T>`, nullability, tuples, tuple-like structs, `Result<T, E>`, fallibility in the type system, `unwrap`/`expect`, error enums, `Error` trait, `Debug` and `Display` traits, packages, crates, library vs. binary targets, `Cargo.toml`, dependencies, `thiserror`, `TryFrom`/`TryInto`, associated types, `Error::source`, trait objects (`dyn Error`), `?` operator, parse-don't-validate pattern.

**Why it matters (AI Systems Engineering):** Type-safe domain modelling and explicit error handling are foundational for building reliable serving layers and CLI tools in Rust, where invalid inputs are rejected at compile time and recoverable errors are encoded in function signatures rather than thrown as exceptions. The `?` operator and structured error types enable clean error propagation through concurrent agent task pipelines and async tool calls without sacrificing safety.

## 06 · Ticket Management

This chapter teaches how to model a ticket management system on top of the `Ticket` struct, introducing collections and data structures needed to store, retrieve, and iterate over multiple tickets. It covers arrays, `Vec`, slices, iterators, combinators, and key-value maps (`HashMap` and `BTreeMap`), along with the trait requirements (`Eq`, `Hash`, `Ord`, `PartialOrd`, `Index`, `IndexMut`) that enable their use. The chapter also introduces lifetimes for tying reference validity to collection scopes, `impl Trait` for returning unnameable iterator types, and the two-state pattern (`TicketDraft` vs. `Ticket`) for modeling entities before and after persistence.

**Rust concepts:** fixed-size arrays (`[T; N]`), `Vec` and growable heap-allocated collections, `Vec::with_capacity` and resizing, `Iterator` and `IntoIterator` traits, `.iter()` and reference iteration, lifetime parameters and lifetime elision, iterator combinators (`map`, `filter`, `filter_map`, `cloned`, `enumerate`, `skip`, `take`, `chain`, `collect`), closures, turbofish syntax, `impl Trait` in return and argument position, slices (`&[T]`, `&mut [T]`), fat pointers, `Deref` coercion, two-state type modeling (`TicketDraft` / `Ticket`), `Index` and `IndexMut` traits, `HashMap`, `BTreeMap`, `Hash` trait, `Eq` trait and reflexivity, `Ord` and `PartialOrd` traits, `Ordering` enum, derive macros for comparison and hashing traits.

**Why it matters (AI Systems Engineering):** Collections, iterators, and key-value stores form the backbone of any serving layer or CLI tool that must batch-process, filter, and route data such as model inference requests, agent task queues, or configuration records. Understanding lifetime-annotated iterators, `impl Trait` returns, and `HashMap`/`BTreeMap` tradeoffs enables building type-safe, zero-cost-abstraction pipelines for concurrent agent workflows or async tool-call dispatch without Python overhead.

## 07 · Threads
This topic provides a comprehensive overview of Rust's concurrency features, starting with basic thread spawning and lifecycle management using `std::thread`. It covers safe data sharing in multithreaded contexts through scoped threads, message passing via mpsc channels (including bounded backpressure), and shared state synchronization using `Arc`, `Mutex`, and `RwLock`. The sections conclude by explaining the `Send` and `Sync` auto traits, which mathematically encode Rust's "fearless concurrency" guarantees across threads.
**Rust concepts:** `std::thread`, `JoinHandle`, `'static` lifetime, `Box::leak`, scoped threads, `std::thread::scope`, mpsc channels (`sync_channel`, `Sender`, `SyncSender`, `Receiver`), interior mutability, `UnsafeCell`, `Mutex`, `RwLock`, `Arc`, `Send` trait, `Sync` trait.
**Why it matters (AI Systems Engineering):** This topic provides the core concurrency primitives needed to build high-performance, parallel data-serving layers or CLI tools in Rust without data races. The channel patterns and locking strategies directly enable safe orchestration of concurrent agent tasks and async tool calls in multi-agent AI systems.

## 08 · Futures
This topic introduces asynchronous programming in Rust as an alternative to OS threads, covering the `async`/`.await` syntax, lazy computation via the `Future` trait, and the architecture of the `tokio` runtime. It details how to manage concurrent tasks using `tokio::spawn`, explains the cooperative nature of Rust's async model (including yield points, cancellation safety, and the critical importance of not blocking the runtime), and contrasts standard synchronous primitives with their async-aware counterparts.
**Rust concepts:** `async fn`, `.await`, `Future` trait, `Poll` (Pending/Ready), async runtimes/executor (`tokio`), `#[tokio::main]`, `#[tokio::test]`, `tokio::spawn`, `JoinHandle`, current-thread vs. multithreaded runtime flavors, work-stealing, `Send` + `'static` bounds on spawned tasks, async blocks (`async move`), state machine transformation, blocking the runtime, `tokio::task::spawn_blocking`, `tokio::sync::Mutex` vs `std::sync::Mutex`, task cancellation (`JoinHandle::abort`), `Drop` for cleanup.
**Why it matters (AI Systems Engineering):** Async Rust and the `tokio` runtime provide the foundational concurrency substrate for the serving layer, enabling high-throughput, non-blocking network interfaces (like HTTP servers) that can juggle massive I/O workloads efficiently. Mastering cooperative scheduling, cancellation, and non-blocking task execution is essential for orchestrating concurrent agent tasks, concurrent LLM API tool calls, and streaming pipelines without starving system resources.
