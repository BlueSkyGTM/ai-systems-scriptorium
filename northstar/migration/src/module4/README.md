# Module 4 — Multi-Agent Systems

The seam's defining material — where Northstar most *is* AI Platform Engineering. The arc: single agent →
runs safely for hours → coordinates with many → **governed as a fleet in production.** The complexity ladder
gates entry: multi-agent only past the single-agent ceiling.

Chapters:

1. **Multi-Agent & Swarms** — why multi-agent, communication protocols (MCP/A2A/ACP/ANP), the four
   primitives, orchestration patterns, debate, failure modes (MASFT).
2. **Autonomous Systems & Operational Safety** — long-horizon agents, durable execution, action budgets,
   kill switches, HITL propose-then-commit, checkpoints, guardrails, Constitutional AI.
3. **Fleet & Loop Engineering** — the production operational layer. "Loops live inside fleets." Registry,
   identity, permissions, inbox-HITL, audit, economics, kill-switch — agent infrastructure as code.
4. **Computer-Use, Coding & Voice** — thin teaching; realized in the artifacts they seed.

roadmap.sh treats multi-agent as a few subtopics; **Northstar makes it a full module** — agent
infrastructure and governance *is* the discipline. The frontier-safety *research/policy* layer (RSP, FSF,
METR, RSI theory) is in the [Antilibrary](../antilibrary.md); the *operational* safety here is kept.
**→ Culminates in Module 7 (Multi-Agent Artifacts).**
