# Module 3 — Agent Foundations

The single-agent spine: `Agent = Reasoning Model + Tool Use + Persistent Memory + Environment Feedback`. You
build one agent that works, runs reliably, and ships. The **complexity ladder** (Gap 2) opens the module —
don't reach for an agent until the problem demands it.

Chapters:

1. **Reasoning & Loops** — ReAct, ReWOO, Reflexion, ToT/LATS, self-refine, planning.
2. **Tools & MCP** — the canonical MCP spine (server → client → transports → security → gateways).
3. **Memory & State** — virtual context (MemGPT), memory blocks (Letta), hybrid stores (Mem0), L1–L3 tiers.
4. **Frameworks & Patterns** — LangGraph/AutoGen/CrewAI/SDKs landscape, selection guide, design patterns.
5. **The Agent Workbench** — the seven surfaces that decide whether an agent ships (instructions, state,
   scope, feedback, verification, review, handoff). Seeds the M6 coding-agent artifact.
6. **TypeScript (entry)** — the break-in set + generics/interfaces, at the point you write your first typed
   tool and MCP contract.

**Multi-agent is Module 4.** Single-agent foundations come first — the same single↔multi seam roadmap.sh's
AI-Agents roadmap uses. **→ Culminates in Module 6 (Agent Artifacts).**
