# kv-cache-sizer

> Back-of-envelope GPU memory for a transformer's KV cache — the number that decides how many
> sequences a card can hold.

Tier: experiment · Status: active

## What it does

Computes `2 · layers · seq_len · kv_heads · head_dim · dtype_bytes · batch` for a few public model
presets, prints per-token and total KV-cache size, and estimates how many concurrent sequences fit on
an 80 GiB card. Shows directly why context length and batch size — not weights — are what blow up at
serve time, and why GQA and lower-precision KV change the math.

## Run it

```
python kv_cache_sizer.py --model llama3-70b --seq-len 8192 --batch 16 --dtype fp16
```

Stdlib only, offline. `--model {llama3-8b,llama3-70b,mistral-7b}`, `--dtype {fp32,fp16,bf16,fp8,int8,int4}`.

## Notes

Numbers are nominal (public spec sheets); real engines add block-padding and fragmentation overhead —
which is exactly what paging reduces. Treat the output as the floor, not the bill.
