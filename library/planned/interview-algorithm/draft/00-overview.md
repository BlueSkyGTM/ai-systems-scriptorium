# Module 1: The Framework

Most candidates walk into an interview carrying a list. A list of stock answers, a list of
projects to name-drop, a list of phrases that test well in mock sessions. The list feels like
preparation. It is preparation for the questions that arrive exactly as rehearsed, which is a
smaller fraction than candidates expect.

This module does not add to the list. It replaces the strategy.

## What This Module Teaches

The Algorithm is a four-step reasoning process for any interview question you have never seen
before. It is not a template you fill in; it is a cognitive tool that tells you what to do in
the first ten seconds after a question lands.

The four steps:

1. **Decompose the question.** What are the literal components? What assumptions are embedded?
   What category does it belong to?
2. **Identify the signal.** What is the interviewer actually trying to learn? Every question
   exists to collect evidence for or against a hiring hypothesis. Name the hypothesis.
3. **Construct the answer.** Build toward the signal, not the question's surface form. Say
   what demonstrates the evidence; omit what does not.
4. **Stress-test it.** Before you finish, check: does this answer actually address the signal?
   Could a weak candidate give the same answer? If so, add specificity.

Two lessons build this module:

1. **Decompose the Question** — how to read what a question is actually asking, beneath the
   words it uses; the signal taxonomy for AI Engineering hiring loops.
2. **Construct the Answer** — how to move from identified signal to a response that
   demonstrates it, with worked examples across behavioral and technical categories.

**What this module feeds forward:** every later module assumes the reader can run steps 1–4
automatically. The behavioral, technical, and systems-design example banks in Modules 3–5 are
the framework loaded into practice; the worked examples are meaningless without the framework
as the organizing lens.
