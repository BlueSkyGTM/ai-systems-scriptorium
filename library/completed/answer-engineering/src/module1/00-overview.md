# Module 1: The Framework

Most candidates walk into an interview carrying a list: stock answers, projects to name-drop, phrases
that tested well in a mock session. The list feels like preparation. It is preparation for the
questions that arrive exactly as rehearsed, which is a far smaller fraction than candidates expect.

This module does not add to the list. It replaces the strategy.

## What This Module Teaches

The Algorithm is a four-step reasoning process for any interview question you have never seen. It is
not a template you fill in; it is the thing you run in the first ten seconds after a question lands.

1. **Decompose the question.** Strip the framing words and find what the question actually names. What
   category does it belong to?
2. **Identify the signal.** Every question exists to collect evidence for or against a hiring
   hypothesis. Name the hypothesis.
3. **Construct the answer.** Build toward the signal, not the question's surface form. Say what
   demonstrates the evidence; cut what does not.
4. **Stress-test it.** Before you finish, check: could a weak candidate give this same answer? If so,
   add specificity.

The failure modes this guards against are not hypothetical. The most common interview mistakes are
structural: jumping to a solution before understanding the problem, answering a question the
interviewer did not ask, describing a happy path with no failure modes. Each is a candidate answering
the surface of the question instead of the signal beneath it. The Algorithm is how you stop doing that.

## The Two Lessons

1. **Decompose the Question** reads what a question is actually asking beneath the words it uses, and
   introduces the signal taxonomy for AI Engineering hiring loops: the four categories every question
   draws from.
2. **Construct the Answer** moves from an identified signal to a response that demonstrates it, with
   worked examples across behavioral, technical, and systems-design questions, and the named
   construction frameworks that scaffold each.

## What This Module Feeds Forward

Every later module assumes you can run these four steps without effort. The behavioral, technical, and
systems-design example banks ahead are the framework loaded into practice; the worked examples are
inert without the framework as the lens that reads them. Install the Algorithm here, and the rest of
the book is repetition until it becomes reflex.
