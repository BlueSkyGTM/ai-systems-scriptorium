# Exercise: Full Algorithm — Signal to Constructed Answer

## Goal

Run the complete four-step Algorithm on five questions from the worked-example bank, write
out constructed answers, and score each against the three construction properties:
specificity, structure, and completeness.

## Why

Decomposition identifies the target. Construction is where you either hit it or miss it. This
exercise closes the gap between knowing what signal a question is collecting and producing an
answer that actually delivers the evidence. The self-scoring step is the crucial habit: you
learn to evaluate your own answers the way an interviewer evaluates them, before it matters.

## Steps

1. Open a new file called `answers-log.md` in this directory. You will write all five answers
   and scores here.

2. Choose five questions. Two must come from the behavioral categories (ownership, conflict,
   failure, influence); one must be a technical depth question; one must be an AI systems-design
   question; the fifth is your choice from any category. If you do not yet have a personal bank
   to draw from, use questions Q1, Q3, Q4, Q7, and Q10 from exercise 01.

3. For each question, run the full Algorithm and record each step:

   **Step 1: Decompose.** Paste your three-part decomposition (literal parse, category,
   primary hypothesis). You can carry this over from exercise 01 if you already did it.

   **Step 2: Identify the signal.** Restate the primary hypothesis in one sentence. Then name
   the evidence the hypothesis requires: what specific things must the answer contain to
   demonstrate the claim?

   **Step 3: Construct the answer.** Write the answer in full, as you would say it. Target
   under two minutes for behavioral questions; under three for technical and systems-design.
   Aim for the 20/60/20 ratio: context, evidence, result.

   **Step 4: Stress-test.** Ask: could a weak candidate give this answer? If yes, underline
   the vague sentence and rewrite it with one specific addition: a number, a named decision,
   a failure mode you caught.

4. After writing and stress-testing the answer, score it on three dimensions. Use a simple
   three-point scale: strong / partial / weak.

   - **Specificity:** Does the answer contain details no weak candidate can supply? Are there
     named techniques, real numbers, or actual decisions, or only general claims?
   - **Structure:** Can you trace the reasoning from situation to action to result? Is the
     ordering logical, or does context and evidence intermix?
   - **Completeness:** Does the answer close the loop on the hypothesis? Could someone extract
     the evidence the primary signal required from this answer?

5. For any dimension scored "partial" or "weak," write one sentence describing what is missing
   and revise the answer to address it.

## Done When

- All five questions have all four Algorithm steps recorded.
- Each answer has a three-dimensional score (specificity, structure, completeness).
- Every "partial" or "weak" score has a revision note and an updated answer.
- At least one answer includes a stress-test revision where you found and fixed a vague sentence.

## Stretch

Record yourself giving one of the five answers out loud, as if in a live interview. Play it
back. Score it on the same three dimensions: does the spoken version hold its specificity and
structure under delivery pressure, or do you revert to the general case when speaking? Note
where the spoken and written answers diverge. Those divergence points are the habits to build
in the deliberate practice sessions in Module 7.
